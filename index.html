🌱 SEED – Social, Economic, Ecologic (Folie 4)

Was ist SEED?

Rahmenmodell für nachhaltige IT-Innovation: Drei Dimensionen müssen gleichzeitig erfüllt sein.

Social Economic Ecologic

Microsoft Beispiel

  • Social: Barrierefreie Software, globale Bildung
  • Economic: Azure-Cloud hilft KMUs Kosten senken
  • Ecologic: CO₂-negativ bis 2030, Meerwasserkühlung

🧠 Collective Intelligence (Folie 14)

Definition

Zusammenspiel von Menschen + Maschinen + Algorithmen, das klüger handelt als jeder Einzelne allein.

Menschen + KI + Plattformen = Collective Intelligence

Beispiel Smart City

  • Sensoren (IoT) liefern Echtzeitdaten
  • KI analysiert Stau & schlägt Ampelschaltung vor
  • Stadtplaner entscheiden per Groupware gemeinsam
  • System lernt mit jeder Entscheidung dazu

🏗️ 3-Schichten-Modell (Folien 67–72)

Die 3 Schichten

  • 🖥️ Präsentationssicht – Benutzeroberfläche
  • ⚙️ Applikationssicht – Geschäftslogik
  • 🗄️ Datenzugriffssicht – Datenbank

Beispiel Online-Banking

  • Nutzer bedient Web-UI (Presentation Layer)
  • Server prüft Überweisungsregeln (Business Logic)
  • Datenbank speichert Transaktionen (Data Layer)
  • + Querschnitt: Sicherheit, Logging, 2FA

🔗 P2P – Peer-to-Peer (Folie 74)

Definition

Alle Teilnehmer (Peers) sind gleichberechtigt – jeder kann gleichzeitig Client UND Server sein. Keine zentrale Instanz.

Peer = Client + Server gleichzeitig

Vorteile P2P

  • Keine Abhängigkeit vom zentralen Server
  • Lastverteilung im Netzwerk
  • Robuster gegen Ausfälle
  • Skaliert automatisch mit Teilnehmerzahl
BitTorrent Blockchain

⚡ Edge Computing (Folie 85)

Definition

Datenverarbeitung findet nicht zentral in der Cloud statt, sondern direkt am Rand des Netzwerks – nah an der Datenquelle.

Verarbeitung → direkt beim Sensor/Gerät

Vorteile & Nachteile

✅ Vorteile:

  • Geringe Latenz
  • Weniger Datenübertragung
  • Höherer Datenschutz
  • Ausfallsicher

❌ Nachteile:

  • Komplexere Architektur
  • Höhere Hardwarekosten
  • Sicherheitsrisiken

📊 Business-IT-Alignment (Folie 83)

Kernidee

Business-Strategie und IT-Strategie müssen aufeinander abgestimmt sein und sich gegenseitig unterstützen.

Business-Strategie ↔ IT-Strategie

Mapping (horizontal)

  • Business-Strategie bestimmt IT-Strategie
  • Org-Struktur bestimmt IS-Infrastruktur

Beispiel E-Commerce

  • Strategie: Online-Vertrieb
  • IT: Webshop, Zahlungsabwicklung
  • Org: Digital Sales Team

🏭 IT-Produktlinie

4 Schritte

Scoping → Domain Engineering → Plattform → Application Engineering
  • 1. Scoping – Domäne eingrenzen (z.B. "Nachhaltige Energiesteuerung für Industriebetriebe")
  • 2. Domain Engineering – Domänenmodell mit Use Cases + Requirements
  • 3. Plattform – Wiederverwendbare gemeinsame Architektur
  • 4. Application Engineering – Kundenwunsch aus Plattform konfigurieren

Case Study Beispiele

  • Plattform: Google Cloud Platform für KI-Energiesteuerung, IBM Blockchain-Framework
  • Application: Continental nutzt Plattform für Reifenproduktion, IBM für Lebensmittelkunde X

Kernaussage

Das Rad nicht neu erfinden = Nachhaltigkeit in der IT!
Wiederverwendbare Plattform spart Zeit, Ressourcen, Geld → direkte Verbindung zu nachhaltigen Betriebsabläufen

📡 Sensor Intelligence + Edge vs. Cloud

Konzept

  • Sensoren erfassen Daten vor Ort (Edge)
  • Vorverarbeitung direkt am Sensor
  • Erst dann relevante Daten → Cloud
  • Spart Bandbreite + ermöglicht Echtzeit

Warum Edge?

Keine Latenz Bandbreite sparen Unabhängig von Internet

3-Schichten + Edge/Cloud (Case Study)

Sensor (Edge) → KI (Cloud) → Steuersignal (Edge)
  • 🔵 Google: Temperatursensor (Edge) → KI berechnet Kühlung (Cloud) → Steuersignal zurück (Edge)
  • 🔵 Continental: Produktionssensor (Edge) → Cloud analysiert Optimierung
  • 🔵 IBM: Kühlketten-Sensor (Edge) → Blockchain (Cloud)

Klausur: 3 Schichten beschreiben + was läuft Edge/Cloud + WARUM

🪞 Digitaler Zwilling + P2P Energie

Digitaler Zwilling

Digitales Abbild der realen Hardware/Prozesse zum Simulieren, Steuern, Optimieren.

  • 🔵 Continental: Zwilling der Reifenproduktion → Materialverbrauch simulieren vor Produktion
  • 🔵 Google: Zwilling der Rechenzentren → KI simuliert Kühlbedarf
  • 🔵 IBM: Zwilling der Lieferkette → Simulation Lieferantenauswahl

P2P im Energienetz (Smart Grid)

  • Alle 3 Schichten auf jedem Knoten lokal
  • Kein zentraler Server
  • Jeder ist gleichzeitig Client + Server

Energie-Beispiel: Privatleute verkaufen Strom an Nachbarn

  • Physisch: normales Stromnetz
  • Digital: P2P Marktplatz-Ebene
  • = Virtuelles Kraftwerk

Hybrid: Autos an Kreuzung reagieren P2P untereinander + Cloud

📋 BPMN / Koordination + Raum-Zeit-Matrix

BPMN Grundlagen

  • Standard für Prozessmodellierung
  • Zustandsübergang → Dienst wird aufgerufen
  • Jeder Schritt nutzt Daten + verändert sie
  • Parallel: beide Schritte fertig → nächster startet

Energiemanagement Continental

  1. Sensor erfasst Verbrauch → Zustand: Erfassung
  2. KI analysiert → Dienst: Analyse-Service
  3. Empfehlung → Dienst: Empfehlungs-Service
  4. Techniker bestätigt → Dienst: Freigabe
  5. Anlage angepasst → Zustand: Abgeschlossen

Raum-Zeit-Matrix (Case Study)

Selbe Zeit (sync)Versch. Zeit (async)
Selber OrtTeam-Workshop nachhaltige ProzesseSchichtübergabe mit Sustainability-Dashboard
Versch. OrtRemote-Monitoring Continental-Werk von HamburgIBM + Lieferant in Asien async. koordinieren

⚖️ LkSG & CSDDD – Lieferkettenrecht

LkSG (Deutschland, 2023)

Lieferkettensorgfaltspflichtengesetz – verpflichtet große Konzerne, Risiken (Menschenrechte, Umwelt) in der gesamten Lieferkette zu prüfen, adressieren und melden.

Merksatz: Nachhaltige Produkte → nachhaltige Lieferketten

🛒 Dienstemarkt & Find-Bind-Invoke

Dienstemarkt Konzept

Plattform, auf der Diensteanbieter ihre Dienste publizieren und Dienstenutzer diese finden und nutzen.

Publish → Find → Bind → Invoke
  • 📢 Publish: Dienst veröffentlichen
  • 🔍 Find: Dienst suchen
  • 🔗 Bind: Schnittstelle aufbauen
  • ▶️ Invoke: Dienst aufrufen

Koordinationsschicht + BPMN

BPMN modelliert den Ablauf (Zustandsübergänge), jede Aktivität ruft einen Dienst auf → Orchestrierung.

Beispiel Klinik:
Aufnahme → Versorgung → Rechnung
→ jeder Schritt ruft automatisch den passenden Dienst auf

Koordination = Nachhaltigkeit in Abläufen

Digitale Souveränität

Die Fähigkeit von Nationen/Unternehmen, eigene digitale Infrastruktur zu kontrollieren – unabhängig von fremden Anbietern.

Schwarz Digit: Deutsche Cloud-Infrastruktur für Mobilitäts- und Gesundheitsanbieter – ein Beispiel für digitale Souveränität.
